Typography

Introducing Carter One: Our Signature Brand Typeface

"Carter One" has been chosen as the primary brand typeface for DailyTube for several compelling reasons. This selection is based on its seamless alignment with the company's design objectives, its remarkable distinctiveness which ensures the brand remains memorable, its potential cultural resonance with the target audience, its adaptability across various media platforms, its consistent adherence to brand values, its reliability in maintaining typography for brand recognition, its readability and scalability, and its technical suitability.

This is Poppins. Our supporting font

"Poppins" is chosen as the supporting font for DailyTube, alongside the primary brand typeface "Carter One," due to its complementary attributes. While "Carter One" fulfills the role of the primary brand typeface, "Poppins" offers a harmonious contrast, enhancing readability and versatility. Its clean and modern sans-serif style provides a balance to "Carter One's" distinctive character, making it suitable for content and user interface elements.

font-family: 'Poppins', Arial, Helvetica, sans-serif;

Data Visualization

To help people make better choices, it's essential to present data in a clear and straightforward way, without unnecessary decorations. So, we've laid out some guidelines for creating meaningful visual data stories. To empower individuals in making informed decisions, we have developed guidelines for creating data-driven visual narratives that focus on DailyTube channel statistics. These visuals encompass aspects such as channel categories, subscriber-to-likes relationships, country-specific insights, top channel performance, and more, presented with user-friendly interactivity and clear design to facilitate data interpretation and decision-making.

Dropdown Select

  • The use of drop-down menus in data visualization allows users to interact with and explore data in a more dynamic and organized way. Instead of overwhelming users with all the information at once, drop-down menus offer a convenient method to select specific data subsets, time periods, or categories, making the data visualization more user-friendly and adaptable. Users can choose what they want to see and focus on, enhancing their overall experience and understanding of the data.

    Examples

Scorecard

  • A scorecard in data visualization provides a quick overview of crucial DailyTube metrics: views, likes, followers, and engagement rate. It simplifies performance assessment for channels and videos, facilitating data-driven decision-making and content strategy.

    Examples

Pie Chart or Donut Chart

  • Pie charts and donut charts are effective data visualization tools used to represent data with distinct categories. Enabling users to grasp the distribution of data categories quickly and efficiently.

    Examples

Scatter Plot Chart or Bubble Chart

  • A scatter plot is a data visualization method that helps depict the relationship between two numerical variables. In the context of DailyTube data, let's use a scatter plot to examine the relationship between the number of followers a DailyTube channel has and the number of likes their videos receive.

    Example

Pivot or Table with Heatmap

  • A table displays the number of DailyTube channels per country, offering a straightforward overview, while a pivot table enhances exploration with interactive sorting and filtering. This feature-rich visualization of DailyTube's channel data can be scrolled for comprehensive analysis, providing an accessible and user-friendly platform for data exploration.

    Examples

Line Graph

  • A line graph is a dynamic visual representation of the yearly view trends of the top 5 DailyTube channels. This tool provides a quick, data-driven overview of how these channels' performance has evolved over time, assisting in making informed content and engagement decisions.

    Examples

Stacked Column Chart

  • A stacked chart in data visualization effectively conveys both individual and cumulative contributions to a total, making it an ideal choice for displaying the quarterly income of the top 5 DailyTube channels. This chart breaks down income by channel within each quarter, allowing viewers to grasp individual and collective earnings, providing valuable insights into their financial performance.

    Examples

Bar Chart

  • A bar chart in data visualization provides a concise way to present the most popular DailyTube categories ranked by followers (horizontally) and identifies the country with the most DailyTubers in the top 100 (vertically). It offers a quick visual comparison of category popularity and top influencer concentrations by country.

    Examples

Buttons

  • Primary

    The "Primary" button features a yellow background with white text. It's clickable, and on hover, its color and background change to create a visual interaction.


    <button class="button primary">Primary</button>

  • Disabled

    The "Disabled" button is visually similar to the primary button, but it's non-clickable, with reduced opacity and a not-allowed cursor.


    <button class="button disabled">Disabled</button>

  • Warning

    The "Warning" button features an orange background, indicating caution. It's a clickable element styled to grab user attention in less critical scenarios.


    <button class="button warning">Warning</button>

  • Link

    The "Link" is a text link that directs users to another page. It's styled in red for unvisited links and blue for visited links, conforming to standard web link conventions. On hover, it changes color to indicate interactivity. It serves as a standard web navigation element.

    This is a sample link

    <a class="one" href="index.html" type="button">This is a sample link</a>

  • Combo Button

    The "Combo Button" combines a primary action button with a dropdown menu for secondary actions. On hover, it reveals the secondary actions, offering a convenient way to present multiple choices within a single button, enhancing user experience and organization.

    Secondary Action 1 Secondary Action 2 Secondary Action 3

    <div class="combo-button"> <button class="combo-primary-action">Combo Button</button> <div class="combo-secondary-actions"> <a href="#" class="combo-secondary-action">Secondary Action 1</a> <a href="#" class="combo-secondary-action">Secondary Action 2</a> <a href="#" class="combo-secondary-action">Secondary Action 3</a> </div> </div>

  • Loading

    The "Loading" button is non-clickable and displays a loading spinner image. It communicates that an operation is in progress and requires patience.

    <button class="button loading" disabled>Loading... <img src="images/loading-spinner.gif" width="50%" alt="Loading Spinner" /></button>

Dialogs

Modals and dialogs are integral components in applications, serving various purposes. They streamline interactions by presenting forms, verification prompts, and contextual information within the current context. These elements enhance user experience, control data integrity, and guide users efficiently, reducing the need for frequent page navigation.

  • Information Dialog

    This button opens an Information Dialog. It provides the user with information, such as success messages, and includes an "OK" button to acknowledge the message.

    Your settings have been saved successfully.

    <button class="button primary" onclick="openDialog('dialog1')">Open Information Dialog</button>

  • Confirmation Dialog

    This button opens a Confirmation Dialog. It asks the user to confirm potentially destructive actions and provides "OK" and "CANCEL" buttons for the user to make a choice.

    Are you sure you want to delete this file? This action cannot be undone.

    <button class="button primary" onclick="openDialog('dialog2')">Open Confirmation Dialog</button>

Forms

Sample User Input Form

  • Text Input

    The following form includes a Basic Text input field for a user to fill out.

    <input type="text" id="text-input" name="text_input" placeholder="Enter text">

  • Required Text Input:

    The following form includes a Text Input with the required attribute, forcing the field to be filled by the user and not skippable.

    <input type="text" id="required-input" name="required_input" placeholder="Enter text" required>

  • Radio Buttons

    Select an Option:

    The following form includes two Radio Buttons. These Radio Buttons cannot be selected simultaneously, forcing the user to select a single option.

    <input type="radio" id="one" name="one" value="one"> <label for="one">One</label> <input type="radio" id="two" name="two" value="two"> <label for="two">Two</label>

  • Checkboxes

    Select Hobbies:

    The following form includes three checkboxes. These checkboxes allow a user to select multiple options at once.

    <input type="checkbox" id="reading" name="hobbies[]" value="reading"> <label for="reading">Reading</label> <input type="checkbox" id="sports" name="hobbies[]" value="sports"> <label for="sports">Sports</label> <input type="checkbox" id="music" name="hobbies[]" value="music"> <label for="music">Music</label>

  • Multi-select (Dropdown)

    The following form includes a drop-down menu. The user will select an option that applies best to them from the provided list.

    <select id="country" name="country"> <option value="usa">USA</option> <option value="canada">Canada</option> <option value="uk">UK</option> <option value="australia">Australia</option> <option value="other">Other</option> </select>

  • Date Input

    The following form includes a Date Input. The Date Input will have a user provide an input in the form of date data that can be easily tracked and displayed.

    <input type="date" id="birthdate" name="birthdate">

  • Email Input

    The following form includes an Email Input. The Email Input will check for an '@' symbol for the data type to be used.

    <input type="email" id="email" name="email">

  • Multiline Text Input

    The following form includes a text area. The text area will allow users to provide multiple lines of text in an organized space.

    <textarea id="comments" name="comments" rows="4" cols="50"></textarea>
